<p class="wp-block-paragraph">The release of Jakarta EE 12 consists of multiple <em>Milestones</em> that are continuously being refined to reflect the current status. In the <a href="https://jakartaee.github.io/platform/minutes/minutes.html" rel="noreferrer noopener" target="_blank">Platform call </a>last week, the platform project set up expectations for a couple of the milestones. First out is Milestone 4, in which the specifications that are part of Jakarta EE Core Profile are expected to show progress in the form of milestone or beta-releases.</p>



<p class="wp-block-paragraph"><strong>M4 Apr 1 to May 15</strong><br /> – REST 5.0 M1 API / spec<br /> – CDI 5.0 Milestone / Beta (currently at Alpha4)<br /> – JSON-P 2.2 M1 API / spec<br /> – JSON-B 3.1 M1 API / spec<br /><strong>M5 May 16 to Jun 30</strong><br /> – content to be defined<br /><strong>M6 Jul 1 to Aug 15</strong><br /> – content to be defined<br /><strong>M7 Aug 15 to Sep 30</strong><br /> – Ideally, we release Core profile here or soon after<br /><strong>M8 Oct 1 to Nov 15</strong><br /> – content to be defined<br /><strong>M9 Jan 1 to Feb 15, 2027</strong><br /> – Finalize all other specifications<br /><strong>M10 Feb 16 to Mar 31, 2027</strong><br /> – Release Platform TCK with ratifying impl</p>



<p class="wp-block-paragraph">The Platform Project will continue to refine this timeline by adding expectations to each milestone. Note that even if a specification is not listed for a specific milestone, nothing prevents it from publishing ahead of time. If the current plan holds, we should be able to release <a href="https://jakarta.ee/specifications/coreprofile/12/" rel="noreferrer noopener" target="_blank">Jakarta EE Core Profile</a> in Q4, 2026, while <a href="https://jakarta.ee/specifications/webprofile/12/" rel="noreferrer noopener" target="_blank">Jakarta EE Web Profile </a>and <a href="https://jakarta.ee/specifications/platform/12/" rel="noreferrer noopener" target="_blank">Jakarta EE Platform</a> will be released in Q1 or Q2 next year. This follows the same pattern as for Jakarta EE 11.</p>

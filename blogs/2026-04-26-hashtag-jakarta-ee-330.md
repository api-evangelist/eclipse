<p class="wp-block-paragraph">Welcome to issue number three hundred and thirty of&nbsp;<a href="https://www.agilejava.eu/category/hashtag-jakartaee/" rel="noreferrer noopener" target="_blank"><em>Hashtag Jakarta EE</em></a>!</p>



<p class="wp-block-paragraph">This week, I was at <a href="https://www.ocxconf.org/event/2026/summary" rel="noreferrer noopener" target="_blank">Open Community eXperience</a> (OCX) in Brussels. I will publish a post about it shortly. This year, OCX unfortunately conflicted with <a href="https://2026.europe.jcon.one/" rel="noreferrer noopener" target="_blank">JCON Europe</a> in Cologne. This was kind of a bummer for me, but these things happen in the busy conference season. Next year, at least these two conferences won’t conflict. The dates for <a href="https://www.ocxconf.org/event/2027/summary" rel="noreferrer noopener" target="_blank">OCX 2027</a> are <strong>April 13-15,</strong> while <a href="https://2027.europe.jcon.one/" rel="noreferrer noopener" target="_blank">JCON Europe 2027</a> will be <strong>May 31-June 3</strong>.</p>

<p class="wp-block-paragraph"><a href="https://www.eclipse.org/" rel="noreferrer noopener" target="_blank">Eclipse Foundation</a> is rolling out GitHub CoPilot Enterprise to all committers and I was so lucky to be in the first batch. It will enable us to automate some of the tasks associated with the JESP (<a href="https://jakarta.ee/about/jesp/" rel="noreferrer noopener" target="_blank">Jakarta EE Specification Process</a>). I set up instructions for CoPilot in the template for Compatibility Requests, so we can use an agent to validate the requests. One of the checklist items for that is to validate that the SHA-256 sums of the TCKs listed match. Definietely something a machine is more capable of getting right than a human. Of course, the actual approval of the request will still need a human in the loop to verify that everything is correct. </p>

I do it from time to time, but I also want to do more, so please send me a link to an article if you want it mentioned here. For example, this article by <a href="https://www.linkedin.com/in/mehmandarov/" rel="noreferrer noopener" target="_blank">Rustam</a> about <a href="https://mehmandarov.com/api-versioning/" rel="noreferrer noopener" target="_blank">API versioning in Java using JAX-RS with Jakarta EE and MicroProfile</a> is an excellent read that I higly recommend that you take a look at.</p>

<p>This error is shown by Open VSX when the Publisher Agreement has <strong>not</strong> been completed. The message is accurate, but easy to misinterpret.</p><h2>TL;DR</h2><ul><li>Contrary to what many assume, this error<strong> is not related</strong> to the "<a href="https://accounts.eclipse.org/user/eca">Eclipse Contributor Agreement</a>" (ECA)</li><li>You must sign the<a href="https://open-vsx.org/user-settings/profile"><strong> Publisher Agreement </strong>on<strong> Open VSX</strong></a><strong> </strong>to publish</li></ul><hr /><h2>Where the confusion comes from</h2><ul><li>A common reaction is to search for this error and follow existing guidance.</li><li>Many sources point to signing the <strong>Eclipse Contributor Agreement</strong>, ECA. The ECA is used when contributing code to Eclipse open source projects: <a href="https://accounts.eclipse.org/user/eca">https://accounts.eclipse.org/user/eca</a></li><li>It ensures that contributions can be legally accepted into Eclipse projects.</li></ul><p><strong>However, it has no impact on your ability to publish extensions on Open VSX.</strong></p><h2>What actually fixes the problem</h2><ul><li>To publish an extension on Open VSX, you must sign a different agreement: <strong>The Publisher Agreement</strong></li><li>This agreement defines the terms under which your extension is distributed via the Open VSX registry.</li></ul><h3>Steps</h3><ol><li>Open your profile on the Open VSX registry:<br /><a href="https://open-vsx.org/user-settings/profile">https://open-vsx.org/user-settings/profile</a></li><li>Log in with GitHub</li><li>Click <strong>Log in with Eclipse</strong></li><li>If you do not yet have an Eclipse Foundation account, create one here:<br /><a href="https://accounts.eclipse.org/user/register">https://accounts.eclipse.org/user/register</a></li><li>Back in your Open VSX profile:<ul><li>Click <strong>Show Publisher Agreement</strong></li><li>Click <strong>Agree</strong></li></ul></li></ol><p>Once this is done, publishing should work as expected.</p><h2>Why this is not obvious</h2><p>From a developer perspective, everything can look correctly configured:</p><ul><li>You are logged in with GitHub</li><li>You have an Eclipse Foundation account</li><li>You may already have signed the Eclipse Contributor Agreement</li></ul><p>Yet publishing still fails.</p><p>The missing step is subtle:</p><ul><li class="ck-list-marker-bold"><strong>The Publisher Agreement must be signed from your Open VSX profile, after linking your Eclipse account.</strong></li></ul><p>Signing agreements on the Eclipse account website alone is not sufficient.</p><h2>ECA vs Publisher Agreement</h2><h3>Eclipse Contributor Agreement</h3><ul><li><strong>Purpose</strong>: contribute code to Eclipse open source projects</li><li><strong>URL</strong>: <a href="https://accounts.eclipse.org/user/eca">https://accounts.eclipse.org/user/eca</a></li><li><strong>Scope</strong>: contribution and IP workflow</li></ul><p><strong>Not used for publishing extensions</strong></p><h3>Publisher Agreement</h3><ul><li><strong>Purpose</strong>: publish extensions on Open VSX</li><li><strong>Location</strong>: your Open VSX profile</li><li><strong>Scope</strong>: distribution terms for extensions</li></ul><p><strong>Required to publish</strong></p><h2>Common pitfalls</h2><h3>“I already signed the contributor agreement”</h3><ul><li>That is expected, but unrelated to publishing.</li><li>You still need to sign the Publisher Agreement.</li></ul><h3>“I don’t have an Eclipse Foundation account”</h3><ul><li>You need one to complete the Publisher Agreement:<br /><a href="https://accounts.eclipse.org/user/register">https://accounts.eclipse.org/user/register</a></li><li><strong>WARNING: Use the same GitHub identity as your Open VSX login </strong>to avoid linking issues.</li></ul><h3>Is it mandatory to sign the Eclipse Contributor Agreement (ECA)?</h3><ul><li><p>No.</p></li><li><p>The ECA is only required if you contribute code to Eclipse open source projects.</p></li></ul><h3>“I don’t see the Publisher Agreement option”</h3><p>Check:</p><ul><li>You are logged in to open-vsx.org with GitHub</li><li>You clicked <strong>Log in with Eclipse</strong></li><li>The accounts are successfully linked</li></ul><p>The option only appears after linking.</p><h3>“I did everything and it still fails”</h3><p>Quick checklist:</p><ul><li>GitHub login active on open-vsx.org</li><li>Eclipse account linked</li><li>Publisher Agreement signed</li><li>Same GitHub identity used throughout</li></ul><h2>Further reading</h2><p><a href="https://github.com/EclipseFdn/open-vsx.org/wiki/Publishing-Extensions">https://github.com/EclipseFdn/open-vsx.org/wiki/Publishing-Extensions</a></p></div>
